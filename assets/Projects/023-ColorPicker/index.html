Color Picker Plugin Documentation

Overview

The Color Picker Plugin is a modular Unreal Engine plugin providing a customizable color picker system built with Widget Blueprints and a C++ backend. This plugin allows developers to integrate color selection tools such as a color wheel, sliders, and input fields into their projects.


Features

  • Modular Widget Blueprint-based UI
  • Color wheel, sliders, and numeric input widgets
  • Runtime-ready C++ backend
  • Optional desktop-level mouse color sampling (Windows only)
  • Easy integration into existing UI systems
  • No external dependencies

Supported Engine Versions

This plugin is compatible with Unreal Engine 5.x versions.


Installation

Method 1: Fab / Epic Games Launcher

  • Open the Epic Games Launcher
  • Go to Fab / Unreal Engine Plugins
  • Download Color Picker
  • Add it to your project
  • Enable the plugin in Edit → Plugins
  • Restart the editor if prompted

Method 2: Manual Installation

  • Copy the ColorPicker folder into:
    • YourProject/Plugins/
  • Open the project
  • Enable the plugin in Edit → Plugins
  • Restart Unreal Engine

Plugin Content Location

All assets are located under:

ColorPicker/Content

This includes Widget Blueprints, Materials, and Textures.


Usage

Using the Widget Blueprints

  • Open your UI Widget Blueprint
  • Add one of the provided widgets:
    • WBP_ColorPicker
    • WBP_ColorWheel
    • Slider widgets
  • Bind to the exposed color output variables or events
  • Apply the selected color to materials, UI elements, or gameplay logic

C++ Integration (Optional)

The plugin exposes C++ functionality for advanced usage. The public headers are located in:

ColorPicker/Source/ColorPicker/Public/

You can include and extend the module in your own C++ code.

Example:

 #include "FL_ColorPicker.h"

Customization

You may modify widget layouts, extend C++ functionality, reuse individual UI components, and customize materials and textures. All assets are editable and modular.


Troubleshooting

  • Ensure the plugin is enabled
  • Restart the editor after enabling
  • Check that the plugin content folder is visible in the Content Browser
  • For C++ projects, regenerate project files if needed
